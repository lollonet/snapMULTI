# ADR-002: Named Pipes (FIFOs) for Audio Routing

## Context

Audio from multiple sources (MPD, Spotify, AirPlay, Tidal) must reach the Snapcast server. Options include: TCP sockets, ALSA loopback, shared memory, or named pipes (FIFOs).

## Decision

Use named pipes (FIFOs) in a shared `/audio/` volume. Each source writes raw PCM to its own FIFO, and Snapcast reads from all of them.

## Consequences

### Positive
- Zero-copy audio path (kernel handles buffering)
- Simple configuration — each source just writes to a file path
- No network overhead (same-host IPC)
- Works with read-only containers (FIFOs are in a shared volume)

### Negative
- FIFO must be held open by a reader — go-librespot needs `exec 3<>/audio/spotify_fifo` workaround
- Permissions require DAC_OVERRIDE capability for cross-user writes
- Fixed audio format — all sources must output 44100:16:2

### Mitigations
- deploy.sh creates FIFOs with 660 permissions and correct ownership
- Entrypoint scripts hold FIFOs open where needed
- Format constraint is acceptable for home audio (CD quality)
