# REQ-031: Graceful Service Recovery

## Description

Services recover automatically from crashes, network interruptions, and dependency failures without manual intervention.

## Recovery Mechanisms

### Container Level
- `restart: unless-stopped`: Docker restarts crashed containers
- Healthcheck retry (3 attempts) before marking unhealthy

### Application Level
- Metadata service: reconnects to snapserver JSON-RPC on connection loss
- go-librespot: re-registers Spotify Connect device after restart
- Shairport-sync: re-advertises AirPlay service via Avahi
- MPD: resumes playback state from `/data/mpd.db`

### Dependency Chain
- Audio sources wait for `snapserver: service_healthy` before starting
- myMPD waits for `mpd: service_healthy`
- Metadata service waits for `snapserver: service_healthy`

## Known Issues

- MPD full NFS scan on container recreate (if `mpd.db` is lost)
- Metadata service may need reconnection after prolonged MPD scan
