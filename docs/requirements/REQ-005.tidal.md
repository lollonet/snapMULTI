# REQ-005: Tidal Connect (ARM Only)

## Description

Tidal Connect allows casting from the Tidal mobile/desktop app to the snapMULTI server. Uses the proprietary `speaker_controller_application` binary, which is ARM-only (Pi 3/4/5). Not available on x86_64.

## Behavior

1. `speaker_controller_application` runs inside tmux for metadata scraping
2. ALSA configured via `tidal-asound.conf` to route audio through speex rate converter to FIFO
3. Audio output goes to `/audio/tidal_fifo` in 44100:16:2 format
4. Device name configurable via `TIDAL_NAME` environment variable
5. `tidal-meta-bridge.sh` scrapes tmux TUI output and writes JSON to `/audio/tidal-metadata.json`
6. `meta_tidal.py` (snapserver plugin) polls the JSON file and forwards to snapserver via JSON-RPC

## Limitations

- ARM only (Pi 3/4/5) — proprietary binary has no x86_64 build
- No album art (TUI does not expose artwork URLs)
- May cause duplicate mDNS entry in Tidal app
- Runs as root (proprietary binary requirement)
- No OAuth — users cast from Tidal app

## Configuration

- `config/tidal-asound.conf`: ALSA routing (speex rate converter + FIFO)
- `TIDAL_NAME`: Custom device name (default: hostname)
- `scripts/tidal/entrypoint.sh`: Container entrypoint
- `scripts/tidal/tidal-meta-bridge.sh`: Metadata scraper

## Dependencies

- Snapserver must be healthy
- D-Bus socket for Avahi/mDNS
- ARM64 host (linux/arm64)
