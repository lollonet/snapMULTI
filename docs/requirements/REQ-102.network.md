# REQ-102: Network Requirements

## Description

All containers run in host networking mode to enable mDNS/Avahi service discovery. This is a fundamental architectural constraint — bridge networking would break multicast DNS.

## Port Allocation

| Port | Service | Protocol | Direction |
|------|---------|----------|-----------|
| 1704 | Snapserver | TCP | Inbound (clients) |
| 1705 | Snapserver | TCP | Inbound (control) |
| 1780 | Snapserver | HTTP | Inbound (Snapweb + JSON-RPC) |
| 4953 | Snapserver | TCP | Inbound (TCP audio source) |
| 6600 | MPD | TCP | Inbound (MPD protocol) |
| 8000 | MPD | HTTP | Inbound (HTTP stream) |
| 8082 | Metadata | WebSocket | Inbound (metadata push) |
| 8083 | Metadata | HTTP | Inbound (artwork + health) |
| 8180 | myMPD | HTTP | Inbound (web UI) |
| 24879 | go-librespot | HTTP | Internal (WebSocket API) |

## mDNS Services

| Service Type | Provider |
|-------------|----------|
| `_snapcast._tcp` | Snapserver |
| `_raop._tcp` | Shairport-sync (AirPlay) |
| `_spotify-connect._tcp` | go-librespot (zeroconf) |
| Tidal Connect | speaker_controller_application |

## Constraints

- Host networking required (no bridge mode)
- No port conflicts with host services
- Multicast must not be blocked by AP isolation
- Firewall must allow inbound on ports listed above
