# REQ-032: Log Management

## Description

All containers use Docker's json-file logging driver with size rotation to prevent disk exhaustion on resource-constrained devices.

## Configuration

```yaml
x-logging: &default-logging
  driver: json-file
  options:
    max-size: "10m"
    max-file: "3"
```

## Behavior

1. Each container's logs are capped at 30MB total (3 x 10MB files)
2. Oldest log file is rotated when size limit is reached
3. Logs accessible via `docker logs <container>` or `docker compose logs`
4. No external log aggregation (designed for single-host deployment)

## Storage Impact

- Maximum log storage: 7 containers x 30MB = 210MB
- Actual usage is typically much lower (most services are quiet)
