# REQ-034: Failure Notifications

## Description

Notify the user when services fail, become unhealthy, or recover. Supports multiple notification channels for home automation integration.

## Proposed Channels

- Webhook (generic HTTP POST)
- MQTT (Home Assistant / Node-RED integration)
- Email (optional, requires SMTP config)

## Proposed Behavior

1. Monitor container healthcheck status changes
2. Debounce notifications (avoid flapping alerts)
3. Send recovery notifications when service returns to healthy
4. Configurable per-service notification rules

## Open Questions

- Standalone notification agent or extend metadata-service?
- Rate limiting to prevent notification storms during restarts?
- Integration with Home Assistant discovery?
