# REQ-041: Now-Playing Display Support

## Description

Display now-playing information (cover art, track title, artist, album) on client devices with screens (e.g., Pi with HDMI, HyperPixel, e-ink). Already partially implemented in the client repo's fb-display container.

## Current State

- Client's `fb-display` container renders cover art to framebuffer
- Metadata-service (server) pushes track info via WebSocket (port 8082)
- Client subscribes with its Snapcast client ID

## Proposed Enhancements

- E-ink display support (low refresh rate, power efficient)
- Small LCD/OLED via SPI/I2C (Waveshare, Adafruit)
- Web-based now-playing page (accessible from any browser)
- Configurable display layout (art only, art+text, text only)

## Open Questions

- Which display hardware to support first?
- Separate container per display type or unified renderer?
- How to handle multiple displays on one client?
