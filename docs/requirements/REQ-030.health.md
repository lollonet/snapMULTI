# REQ-030: Container Health Monitoring

## Description

Every container has a Docker healthcheck that verifies the service is responsive. Docker's restart policy (`unless-stopped`) ensures automatic recovery.

## Healthchecks

| Service | Method | Interval | Start Period |
|---------|--------|----------|-------------|
| snapserver | `pidof snapserver` | 60s | 15s |
| shairport-sync | `pgrep + test FIFO` | 60s | 10s |
| librespot | `pgrep + wget API` | 60s | 15s |
| mpd | `nc 127.0.0.1 6600` | 60s | 60s (300s NFS) |
| mympd | `wget :8180/` | 60s | 60s (300s NFS) |
| metadata | `urllib :8083/health` | 60s | 10s |
| tidal-connect | `pgrep tidal` | 60s | 10s |

## Recovery

- `restart: unless-stopped` on all containers
- 3 retries before marking unhealthy
- 10s timeout per healthcheck probe
- Service dependency chain: snapserver must be healthy before sources start

## Dependencies

- Docker Engine healthcheck support
