# REQ-101: Resource Constraints

## Description

Every container has explicit memory limits and CPU quotas to prevent any single service from starving others. Limits are configurable via environment variables and auto-detected by `deploy.sh`.

## Resource Profiles

### Minimal (Pi 3, < 2GB RAM)
| Service | Memory Limit | CPU Limit |
|---------|-------------|-----------|
| snapserver | 128M | 0.5 |
| shairport-sync | 64M | 0.3 |
| librespot | 64M | 0.3 |
| mpd | 128M | 0.5 |
| mympd | 64M | 0.3 |
| metadata | 64M | 0.3 |
| tidal-connect | 64M | 0.3 |

### Standard (Pi 4 2GB)
Per-service defaults in docker-compose.yml: 128-256MB memory, 0.5-1.0 CPU.

### Performance (Pi 4 4GB+, Pi 5)
Up to 512MB memory, 2.0 CPU per service.

## Configuration

Environment variables per service:
- `{SERVICE}_MEM_LIMIT` — cgroup memory hard limit
- `{SERVICE}_MEM_RESERVE` — guaranteed minimum memory
- `{SERVICE}_CPU_LIMIT` — CPU time fraction

## Dependencies

- Docker Engine with cgroup v2 support
- `deploy.sh` for auto-detection (or manual `.env` configuration)
