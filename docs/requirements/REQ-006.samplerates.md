# REQ-006: Configurable Audio Sample Rates

## Description

Currently all sources are fixed at 44100:16:2 (CD quality). Higher sample rates (48kHz, 96kHz) would benefit sources like Tidal HiFi Plus (up to 96kHz/24-bit), Spotify (48kHz native), and hi-res local files via MPD.

## Proposed Behavior

1. Configurable sample rate per source in snapserver.conf
2. Supported rates: 44100, 48000, 96000 Hz
3. Bit depths: 16-bit, 24-bit
4. All sources in a stream group must use the same format
5. ALSA rate conversion where source native rate differs

## Affected Components

- `config/snapserver.conf`: source format parameters
- `config/mpd.conf`: output format
- `config/tidal-asound.conf`: speex rate converter target
- `config/go-librespot.yml`: output format
- `config/shairport-sync.conf`: output format
- `scripts/deploy.sh`: format validation

## Open Questions

- Should format be global or per-source?
- How to handle format mismatch between grouped sources?
- Impact on CPU usage at higher sample rates on Pi 4
