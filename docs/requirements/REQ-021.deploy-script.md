# REQ-021: Deploy Script for Any Linux Host

## Description

`scripts/deploy.sh` automates server deployment on any Linux host (Pi or x86_64). It detects hardware, generates `.env`, creates directories and FIFOs, installs Docker if needed, and starts all services.

## Behavior

1. Detects CPU architecture (arm64 vs amd64) and RAM
2. Selects hardware profile (minimal/standard/performance) based on available RAM
3. Creates directory structure (`audio/`, `config/`, `data/`, `mpd/`, `mympd/`, `artwork/`)
4. Creates named pipes (FIFOs) with 660 permissions
5. Generates `.env` from `.env.example` with detected values
6. Detects NFS mounts and adjusts `MPD_START_PERIOD` accordingly
7. Validates Tidal ALSA config on ARM systems
8. Installs Docker and Docker Compose if not present
9. Runs `docker compose pull && docker compose up -d`

## Profiles

| Profile | RAM Threshold | Per-Service Limits |
|---------|--------------|-------------------|
| minimal | < 2GB | 128MB RAM, 0.5 CPU |
| standard | 2-4GB | 256MB RAM, 1.0 CPU |
| performance | > 4GB | 512MB RAM, 2.0 CPU |

## Dependencies

- Debian/Ubuntu-based Linux distribution
- Internet connectivity (for Docker install and image pull)
- Root or sudo access
