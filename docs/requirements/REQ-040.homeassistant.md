# REQ-040: Home Assistant Integration

## Description

Expose snapMULTI services as Home Assistant entities via MQTT auto-discovery, enabling playback control, volume adjustment, and source switching from HA dashboards and automations.

## Proposed Entities

| Entity | Type | Controls |
|--------|------|----------|
| media_player.snapmulti_mpd | media_player | play/pause, volume, track info |
| media_player.snapmulti_spotify | media_player | play/pause, volume, track info |
| sensor.snapmulti_clients | sensor | Connected client count |
| binary_sensor.snapmulti_health | binary_sensor | Overall system health |

## Proposed Architecture

- MQTT broker (Mosquitto) as optional container
- Bridge service publishes state changes to MQTT
- HA discovers entities via `homeassistant/` MQTT prefix

## Open Questions

- Require user to provide external MQTT broker or bundle Mosquitto?
- Scope: read-only status or full playback control?
- How to handle source switching (Snapcast groups)?
