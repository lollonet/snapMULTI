# REQ-004: TCP Audio Input

## Description

Snapserver accepts raw PCM audio via TCP socket, allowing any application capable of sending audio data over the network to act as a source. This provides an open integration point for custom audio pipelines.

## Behavior

1. Snapserver listens on TCP port 4953 for incoming PCM audio
2. Audio format must match 44100:16:2 (same as all other sources)
3. Multiple TCP connections can coexist but each is a separate stream
4. No authentication required (host networking limits exposure to LAN)

## Configuration

- Defined in `config/snapserver.conf` as a `tcp` source type
- Port configurable via snapserver source parameters

## Dependencies

- Snapserver must be running and healthy
- Sending application must be on the same network (host networking)
