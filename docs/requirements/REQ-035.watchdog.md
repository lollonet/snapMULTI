# REQ-035: Hardware Watchdog

## Description

Enable the Raspberry Pi hardware watchdog to automatically reboot the system if it becomes unresponsive. Prevents the need for manual power cycling of a headless appliance.

## Proposed Behavior

1. Enable `bcm2835_wdt` kernel module on Pi
2. Configure `systemd` watchdog with appropriate timeout (60s)
3. Docker daemon health feeds the watchdog
4. If watchdog is not fed within timeout, Pi reboots

## Configuration

- `deploy.sh` or `firstboot.sh` enables watchdog on Pi hardware
- Timeout configurable via environment variable
- Disabled on x86_64 (assumes managed infrastructure)

## Open Questions

- Should individual container failures trigger reboot or just Docker daemon hangs?
- Appropriate timeout value for NFS-heavy workloads (MPD scan)?
