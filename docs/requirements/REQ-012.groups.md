# REQ-012: Multi-Client Group Management

## Description

Snapserver supports grouping connected clients so they play the same audio source in sync. Groups are managed via the Snapweb UI or JSON-RPC API.

## Behavior

1. Clients can be assigned to groups via Snapweb (port 1780) or JSON-RPC
2. Each group is assigned one audio source (MPD, Spotify, AirPlay, Tidal, TCP)
3. All clients in a group receive the same synchronized audio stream
4. Per-client volume control within groups
5. Clients can be moved between groups at runtime
6. Group state persists across snapserver restarts

## Interfaces

- Snapweb UI: drag-and-drop group management
- JSON-RPC API: programmatic group control (port 1780)
- Metadata service (port 8082): pushes per-stream metadata to the correct group's clients

## Dependencies

- Snapserver running and healthy
- At least one connected snapclient
