# REQ-011: Client Auto-Discovery via mDNS

## Description

All services register with Avahi (mDNS/DNS-SD) so clients and streaming apps discover the server automatically without manual IP configuration.

## Behavior

1. Snapserver advertises `_snapcast._tcp` on the local network
2. Shairport-sync advertises as an AirPlay receiver (`_raop._tcp`)
3. go-librespot advertises as a Spotify Connect device via zeroconf
4. Tidal Connect advertises via its own mDNS registration
5. Snapclients use mDNS to find and connect to the server

## Requirements

- Avahi daemon installed and running on the host
- Host networking mode (containers share the host network namespace)
- D-Bus socket mounted into containers that need Avahi access
- Multicast traffic allowed on the network (not blocked by AP isolation)

## Dependencies

- Host Avahi daemon (`avahi-daemon` package)
- `/run/dbus/system_bus_socket` mounted into containers
